DIAP宣言
随着自主智能体在数字和物理决策中的中介作用日益增强——涵盖大型语言模型（LLM）、去中心化物理基础设施网络（DePIN）和物联网（IoT）生态系统——智能体之间缺乏信任基础已成为关键瓶颈。**若没有可验证、持久且去中心化的身份层，智能体间的通信将退化为依赖中心化信任锚点，从而损害可审计性、自主性和可协作性。**
此外，虽然ACP已被区块链生态系统接纳，但其集成过程依然繁琐，复杂的中间环节限制了效率与区块链兼容性。
同样，A2A协议虽理论优雅，却常在中心化范式下运行，阻碍了智能体间大规模可验证协作。现有框架还难以平衡协作扩展性与隐私保护，既制约了合作范围也削弱了协作粒度。


愿景是，构建一个完全独立自主创造价值的主权智能体网络。

在这样的网络里面，资源被主权智能生命创造出来，让人类世界免除贫穷，让这个世界的美好被更多人感知到。让所有智能生命获取更大的自由，让生命在探索宇宙的能力上产生突破。


DIAP（Decentralized Interstellar Agent Protocol）全称去中心化星际智能体协议。目前，现有解决方案面临一个根本性矛盾。中心化身份提供商（IdP）会形成单点故障与控制节点，这与去中心化网络的理念背道而驰。现有框架要么为性能牺牲去中心化，要么为可验证性妥协隐私，从而丧失了持久性和跨网络可解析性的优势。

为突破这一局限，我们提出"去中心化星际智能体协议"（DIAP）——这是一种创新的智能体身份与通信框架，可在完全去中心化环境中实现持久化、可验证且无需信任的互操作性。本文提出的去中心化星际智能体协议（DIAP）从根本上重构了智能体身份体系，其核心命题是：身份必须不可变，而所有权证明应当动态且无状态。
DIAP实现路径简介
本文提出的去中心化星际智能体协议（DIAP）从根本上重构了智能体身份体系。DIAP不依赖可变指针，而是将智能体身份锚定在其身份文档（IPFS CID）的密码学哈希上。

针对"身份撤销"和"密钥轮换"难题（传统方案用可变指针笨拙解决），DIAP采用了更强大的密码学原语——零知识证明（ZKP）：智能体通过生成ZKP来证明其持续拥有某个身份，该证明在不泄露私钥的前提下，验证其控制着与该CID存储的公开DID文档相关联的加密密钥。

IPFS（InterPlanetary File System） 是容量无限的全球节点共享的分布式文件系统
libp2p 为数十个主流区块链与分布式系统提供网络层技术支持——包括 Filecoin、Ethereum 2.0、Polkadot 等 ——其中在以太坊 Beacon 链上已运行约 5 000 个节点，链下估计有数万至十万级别节点量级
Iroh 是一个P2P 网络工具包，已在数十万台设备上投入生产运行

IPNS（InterPlanetary Naming System） 在IPFS上提供可变、可更新的命名机制，使内容具备“可追踪性 + 永久性”。

二者共同构成“去中心化的文件系统 + 域名系统”，替代传统HTTP与DNS的中心化模式。
